# Лабораторная работа № 3  
## Использование сигнатурного анализатора для эксплуатационного обслуживания мобильных систем  

## Цель занятия  
1. Научить студентов применять сигнатурный анализатор для диагностики и оптимизации мобильных приложений.  
2. Развить навыки работы с Python‑скриптами, взаимодействия с API анализатора и интерпретации полученных данных.  

## Задачи занятия  
- Ознакомить с принципами работы сигнатурных анализаторов.  
- Научить формировать запросы к API и обрабатывать ответы.  
- Практически применить инструменты анализа для реального мобильного проекта.  
- Оценить полученные результаты и предложить меры по улучшению.  

## План занятия (90 мин.)  

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Теоретическая часть: принципы сигнатурного анализа — 20 мин  
- Практическая работа: настройка и запуск анализатора — 25 мин  
- Обсуждение результатов и их интерпретация — 10 мин  
- Итог занятия — 10 мин  

## Содержание этапов  

- **Организационный момент** – проверка присутствия, объявление темы, краткое разъяснение цели и плана.  
- **Актуализация знаний** – быстрый опрос по предыдущим лабораторным работам, закрепление основ Python и работы с внешними сервисами.  
- **Теоретическая часть** – обзор архитектуры сигнатурного анализатора, типы сигнатур, алгоритмы сопоставления, основные метрики производительности.  
- **Практическая работа** – студенты в парах запускают скрипт, подключают его к тестовому мобильному приложению, получают отчёт о сигнатурных несовпадениях и рекомендуемых исправлениях.  
- **Обсуждение результатов** – коллективный разбор полученных отчётов, выявление общих проблем и обсуждение возможных оптимизаций.  
- **Итог занятия** – резюмирование ключевых моментов, ответы на вопросы, раздача чек‑листов по работе с анализатором.  

## Оборудование и материалы  
- Компьютеры с установленной IDE (PyCharm/VS Code)  
- Доступ к интернету и API сигнатурного анализатора  
- Проектор/экраны для демонстрации кода и отчётов  
- Копии учебных материалов: «Сигнатурный анализатор: Руководство пользователя», «Python для мобильных систем»  
- Задание лабораторной работы в формате PDF  

## Контроль и оценивание  
- Вопросы на закрепление: 3‑4 вопроса после теоретической части.  
- Практическое задание: выполнение скрипта, получение отчёта, отправка кода и логов преподавателю.  
- Краткая оценка: 5‑балльная шкала за корректность выполнения, полноту отчёта и качество комментариев в коде.  

## Домашнее задание  
1. Написать скрипт, который автоматически сравнивает два отчёта сигнатурного анализа и выделяет новые несовпадения.  
2. Подготовить краткое (до 300 слов) резюме о том, какие сигнатуры оказались наиболее критичными для выбранного мобильного приложения и почему.
