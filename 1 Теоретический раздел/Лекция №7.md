# Сущность метода сигнатурного анализа, его достоинства. Приборы для проведения сигнатурного анализа

## Цели и задачи лекции
- Понять основные принципы сигнатурного анализа кода.
- Оценить преимущества и ограничения данного метода.
- Ознакомиться с инструментами, применяемыми в сигнатурном анализе.
- Научиться применять сигнатурный анализ в реальных задачах по Python.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Сигнатура | Уникальное описание функции или метода, включающее имя, типы параметров и возвращаемое значение. |
| Сигнатурный анализ | Метод обнаружения и сопоставления сигнатур в исходном коде для выявления закономерностей, ошибок и потенциальных уязвимостей. |
| Токен | Минимальная единица синтаксического анализа, например, слово, число или символ. |
| Статический анализ | Проверка кода без его выполнения, основанная на синтаксическом и семантическом анализе. |
| Инструмент сигнатурного анализа | Программа или библиотека, реализующая алгоритмы поиска и сопоставления сигнатур. |
| Фильтр сигнатур | Набор правил, определяющих, какие сигнатуры считать релевантными для анализа. |
| Паттерн | Шаблон, описывающий структуру кода, который можно использовать в сигнатурном анализе. |
| Декоратор | Функция, принимающая другую функцию и возвращающая модифицированную версию. |
| Метаданные | Информация, сопутствующая коду, такая как типы данных, аннотации или документация. |
| Интеграция | Внедрение инструмента сигнатурного анализа в процесс разработки и CI/CD. |

## Основное содержание

### 1. Введение в сигнатурный анализ (10 мин)
- История и развитие метода.
- Сравнение с другими техниками статического анализа.
- Почему сигнатурный анализ важен для Python-разработчиков.

### 2. Механизмы работы сигнатурного анализа (20 мин)
- Парсинг кода и построение абстрактного синтаксического дерева (AST).
- Выделение токенов и построение сигнатурных шаблонов.
- Алгоритмы сопоставления: точное и частичное совпадение.
- Обработка дженериков и аннотаций типов.

#### Пример: построение сигнатуры функции
```python
def add(a: int, b: int) -> int:
    return a + b
```
Сигнатура: `add(int, int) -> int`

### 3. Достоинства сигнатурного анализа (15 мин)
- **Универсальность** – работает с любым кодом, не требуя его выполнения.
- **Раннее обнаружение ошибок** – выявление несовпадений типов, недостающих аргументов.
- **Поддержка больших проектов** – автоматическое обнаружение дублирования и неиспользуемого кода.
- **Интеграция с IDE** – подсказки и автодополнение.

### 4. Ограничения и риски (10 мин)
- Плохая точность при динамической типизации.
- Ограниченное понимание контекста выполнения.
- Возможность ложных срабатываний (false positives).

### 5. Приборы для сигнатурного анализа (20 мин)
| Инструмент | Краткое описание | Применение в Python |
|------------|------------------|---------------------|
| **pylint** | Статический анализ кода с поддержкой сигнатур. | Проверка типов и стиля. |
| **mypy** | Проверка типов с помощью аннотаций. | Сигнатурный анализ и статическая типизация. |
| **pyright** | Быстрый анализ типов, поддержка IntelliSense. | Интеграция в VS Code. |
| **Bandit** | Поиск уязвимостей, использующий сигнатурные шаблоны. | Безопасность кода. |
| **pyflakes** | Минимальный анализ синтаксиса и ошибок. | Быстрая проверка. |
| **Radon** | Метрики кода, включая сложность и количество сигнатур. | Оценка качества кода. |

#### Практический пример: проверка типов с mypy
```bash
# Установить mypy
pip install mypy

# Файл example.py
def greet(name: str) -> None:
    print(f"Hello, {name}")

greet(42)  # Ошибка mypy: тип str ожидается, но передан int
```

  ![Mypy Flowchart](https://example.com/mypy_flow.png) *(пример, не вставляется в Obsidian)*
- **Код с декоратором и сигнатурой**  
  ```python
  from functools import wraps
  from typing import Callable, Any

  def log_call(func: Callable) -> Callable:
      @wraps(func)
      def wrapper(*args, **kwargs) -> Any:
          print(f"Calling {func.__name__} with {args} {kwargs}")
          return func(*args, **kwargs)
      return wrapper

  @log_call
  def multiply(x: int, y: int) -> int:
      return x * y
  ```
