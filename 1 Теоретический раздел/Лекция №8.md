## Ключевые понятия  
| Термин | Определение |
|--------|-------------|
| Схемный контроль | Верификация работоспособности электрических/логических схем без физического подключения к устройству. |
| Алгоритм тестирования | Последовательность шагов, определяющих порядок подачи сигналов и чтения результатов. |
| Тестовый паттерн | Набор входных значений, применяемый для проверки конкретного участка схемы. |
| Тестовый стенд | Оборудование или программная среда, симулирующая работу схемы. |
| Тестовая документация | Набор документов, описывающих тестовые сценарии, результаты и критерии принятия. |
| Модуль тестирования | Независимая часть тестового стенда, реализующая конкретный тип проверки. |
| Критерий прохождения | Условие, при котором тест считается успешным. |
| Лог тестирования | Запись всех событий, произошедших во время теста. |
| Симулятор | Программное обеспечение, имитирующее работу схемы в разных режимах. |
| Интеграционный тест | Проверка взаимодействия нескольких модулей схемы. |

## Примеры и иллюстрации  

### Пример схемы и тестового паттерна  
- Схема: простая логическая цепочка (AND → OR).  
- Паттерн: входы `[1,0] → ожидаемый выход 0`.  

```python
def and_gate(a, b): return a and b
def or_gate(a, b): return a or b

def simulate_circuit(inputs):
    a, b = inputs
    and_out = and_gate(a, b)
    return or_gate(and_out, a)
```
