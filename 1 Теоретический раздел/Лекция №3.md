# Надежность системы. Факторы, влияющие на надежность. Методы повышения надежности. Достоверность информации

## Цели и задачи лекции
- Понять, что такое надежность и как она измеряется в программных системах.
- Изучить ключевые факторы, влияющие на надежность программного обеспечения.
- Познакомиться с методами повышения надежности и их применением в Python.
- Осознать роль достоверности информации в контексте надежности.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Надежность (Reliability) | Способность системы выполнять заданные функции без отказов в течение заданного периода времени. |
| Уровень отказов (Failure Rate) | Среднее число отказов в единицу времени, часто выражается как MTBF (Mean Time Between Failures). |
| Доступность (Availability) | Доля времени, когда система доступна для использования, учитывая MTBF и MTTR (Mean Time To Repair). |
| Тестирование на надёжность (Reliability Testing) | Процедуры, направленные на выявление и устранение ошибок до эксплуатации. |
| Валидация данных (Data Validation) | Проверка корректности и целостности данных, используемых системой. |
| Фактор среды (Environmental Factor) | Внешние условия (температура, влажность, электромагнитные помехи), влияющие на работу оборудования. |
| Фактор человеческий (Human Factor) | Ошибки пользователя, администрирования или разработчика, влияющие на надёжность. |
| Фактор отказоустойчивости (Fault Tolerance) | Способность системы продолжать работу при наличии ошибок. |
| Фактор избыточности (Redundancy) | Использование дублирующих компонентов для обеспечения непрерывности работы. |
| Достоверность информации (Data Integrity) | Гарантия того, что данные не были изменены без разрешения и сохраняют свой оригинальный смысл. |

## Основное содержание

### 1. Введение в надежность (5 минут)
- Определение и важность надёжности в IT.
- Связь надёжности с безопасностью и доступностью.

### 2. Модели и метрики надежности (15 минут)
- MTBF, MTTR, Availability = MTBF / (MTBF + MTTR).
- Пример расчёта доступности для сервера.

```python
MTBF = 2000  # часов
MTTR = 5     # часов
availability = MTBF / (MTBF + MTTR)
print(f"Availability: {availability:.4f}")
```

### 3. Факторы, влияющие на надёжность (20 минут)
- **Технические факторы**: устаревшее ПО, несовершенные алгоритмы, недостаточное тестирование.
- **Человеческие факторы**: ошибки в коде, неверные настройки.
- **Экологические факторы**: температура, влажность, электромагнитные помехи.
- **Инфраструктурные факторы**: отказ оборудования, сетевые проблемы.

#### Таблица влияния факторов
| Фактор | Пример влияния | Как уменьшить |
|--------|----------------|---------------|
| Ошибки кода | Null‑пойнтеры | Код-ревью, тесты |
| Перегрузка сети | Потеря пакетов | QoS, балансировка нагрузки |
| Неправильный таймаут | Deadlock | Тестирование таймаутов |

### 4. Методы повышения надёжности (25 минут)
#### 4.1 Тестирование (10 минут)
- Unit‑тесты, интеграционные тесты, нагрузочные тесты.
- Использование `pytest` и `unittest`.

```python
def add(a, b):
    return a + b

def test_add():

```

#### 4.2 Исключения и обработка ошибок (5 минут)
- `try/except` блоки, логирование ошибок.
- Пример обработки исключения при работе с файлом.

```python

    with open('data.txt', 'r') as f:
        data = f.read()
except FileNotFoundError:
    print("Файл не найден")
```

#### 4.3 Избыточность и резервирование (5 минут)
- Репликация БД, балансировка нагрузки, кластеризация.

#### 4.4 Валидация данных (5 минут)
- Проверка входных данных, использование `pydantic` или `marshmallow`.

```python
from pydantic import BaseModel, ValidationError

class User(BaseModel):
    name: str
    age: int

try:
    user = User(name="Alice", age="twenty")
except ValidationError as e:
    print(e)
```

### 5. Достоверность информации (10 минут)
- Принципы проверки целостности: контрольные суммы, хеш-функции, цифровые подписи.
- Практика: проверка целостности скачанного файла.

```python
import hashlib

def check_hash(file_path, expected_hash):
    hash_md5 = hashlib.md5()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(4096), b""):
            hash_md5.update(chunk)
    return hash_md5.hexdigest() == expected_hash
```

## Выводы по лекции
- Надежность критична для устойчивой работы программных систем.
- Факторы надёжности разнообразны, от технических до человеческих.
- Методы повышения надёжности включают тестирование, обработку ошибок, избыточность и валидацию.
- Достоверность информации обеспечивает корректность и целостность данных, что напрямую влияет на надёжность.
