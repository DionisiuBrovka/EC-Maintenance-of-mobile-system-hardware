## Теоретическая часть

| Понятие | Определение | Примечание |
|---------|-------------|------------|
| Оперативная память (RAM) | Быстрый тип памяти, используемый для хранения данных и инструкций, которые активно обрабатываются процессором. | Жизненно важна для производительности системы. |
| Виртуальная память | Механизм, позволяющий операционной системе использовать дисковое пространство как «расширение» физической памяти. | Позволяет запускать программы, превышающие объём RAM. |
| Фрагментация памяти | Разделение доступной памяти на неравные блоки, что может снизить эффективность использования. | Может быть внешней (разные размеры блоков) и внутренней (неиспользуемое место внутри блока). |
| Аллокация памяти | Процесс выделения области памяти для объектов во время выполнения программы. | В Python управляется сборщиком мусора. |
| Сборщик мусора (Garbage Collector) | Механизм, автоматически освобождающий неиспользуемые объекты в памяти. | В CPython реализован подсчётом ссылок и циклическим сбором. |
| Операционная система (ОС) | Программный комплекс, управляющий аппаратными ресурсами и предоставляющий сервисы приложениям. | Важна для контроля доступной памяти. |
| Память кэша | Быстрый тип памяти, расположенный близко к процессору, используемый для хранения часто обращаемых данных. | Улучшает производительность за счёт снижения времени доступа. |
| Файл подкачки (swap) | Дисковый файл, используемый как «дополнительная» память при нехватке RAM. | Доступ к нему медленнее, чем к физической памяти. |
| Оперативный менеджер памяти | Компонент ОС, отвечающий за распределение и освобождение памяти между процессами. | В Linux – `vmstat`, `free`, `top`. |
| Профилирование памяти | Измерение потребления памяти программой для оптимизации. | В Python – модули `tracemalloc`, `memory_profiler`. |

## Ход выполнения работы

1. **Подготовка окружения**  
   - Откройте терминал/консоль.  
   - Убедитесь, что Python 3.8+ установлен.  
   ```bash
   python --version
   ```  
   - Установите необходимые пакеты:  
   ```bash
   pip install psutil memory_profiler
   ```

2. **Получение общей информации о памяти**  
   - Запустите скрипт `mem_info.py`, который использует библиотеку `psutil`.  
   ```python
   import psutil

   vm = psutil.virtual_memory()
   print(f"Total: {vm.total / (1024**3):.2f} GB")
   print(f"Available: {vm.available / (1024**3):.2f} GB")
   print(f"Used: {vm.used / (1024**3):.2f} GB")
   print(f"Free: {vm.free / (1024**3):.2f} GB")
   ```
   - Сравните значения с тем, что отображает `free -h` в Linux или `Task Manager` в Windows.

3. **Тестирование памяти с помощью Python‑скрипта**  
   - Создайте файл `memory_test.py`.  
   - Напишите функцию, создающую большие списки и измеряющую потребление памяти.  
   ```python
   import tracemalloc
   import time

   def big_list(n):
       return [i for i in range(n)]

   tracemalloc.start()
   start = time.time()
   lst = big_list(10**7)  # 10 млн элементов
   current, peak = tracemalloc.get_traced_memory()
   end = time.time()
   tracemalloc.stop()

   print(f"Time: {end - start:.2f}s")
   print(f"Current memory usage: {current / (1024**2):.2f} MB")
   print(f"Peak memory usage: {peak / (1024**2):.2f} MB")
   ```
   - Запустите скрипт и зафиксируйте результаты.

4. **Анализ фрагментации памяти**  
   - Используйте `tracemalloc` для отслеживания «списков» и их размеров.  
   - Сравните результаты с использованием `gc.collect()` для принудительной сборки мусора.  
   ```python
   import gc
   gc.collect()
   ```

5. **Сравнение с системными утилитами**  
   - Запустите `top` (Linux) или `htop` и наблюдайте за колонкой `%MEM`.  
   - Сравните показатели с тем, что выводит ваш скрипт.

6. **Документирование результатов**  
   - Запишите в отчёт: время выполнения, текущую и пиковую память, сравнение с системными данными.  
   - Включите графики (если есть).
