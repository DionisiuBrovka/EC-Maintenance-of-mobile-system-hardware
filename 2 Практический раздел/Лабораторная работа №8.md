# Лабораторная работа № 8  
Восстановление информации на жестком диске и Flash‑накопителе  

## Цель и задачи  
1. Научиться использовать инструменты восстановления данных.  
2. Овладеть принципами работы файловых систем и механизмов удаления файлов.  
3. Выполнить практическое восстановление удалённых файлов на примере HDD и USB‑накопителя.  
4. Проанализировать результаты и понять ограничения восстановительных процессов.  

## Теоретическая часть  

| Понятие | Описание |
|---------|----------|
| **Файловая система** | Система организации данных на носителе (NTFS, FAT32, exFAT). |
| **Сегмент** | Блок данных фиксированного размера, отведенный под файл. |
| **Метаданные** | Информация о файле (имя, дата, размер, список сегментов). |
| **Том (volume)** | Физический или логический раздел, содержащий файловую систему. |
| **Память Flash** | Накопитель с твердотельным хранилищем, использующий NAND‑чипы. |
| **TRIM** | Команда, отправляемая ОС, чтобы удалить данные из NAND‑чипов. |
| **Пагинация** | Перемещение данных между основным и виртуальным хранилищем. |
| **Кластер** | Минимальный единичный блок, выделяемый файловой системой. |
| **Фрагментация** | Распределение сегментов файла по разным кластерам. |
| **Алгоритм восстановления** | Процесс чтения сырых данных и восстановления файлов по метаданным и паттернам. |

## Ход выполнения работы  

1. **Подготовка среды**  
   - Скачайте и установите программы `TestDisk` и `PhotoRec`.  
   - Подготовьте два USB‑накопителя: один с файловой системой NTFS, другой – exFAT.  
   - На каждом накопителе создайте папку `lab8` и поместите туда несколько файлов (текст, изображения).  

2. **Удаление файлов**  
   - Удалите все файлы из папки `lab8` через проводник.  
   - Проверьте, что файлы перемещены в корзину, и очистите корзину, чтобы удалить ссылки.  

3. **Запуск TestDisk**  
   - Откройте терминал/командную строку и выполните `testdisk`.  
   - Выберите диск, создайте новый лог-файл, выберите тип файловой системы (`NTFS`/`exFAT`).  
   - Выполните анализ (`Analyze` → `Quick Search`).  
   - Если найдено разделы, выберите их и выполните `Reconstruct`.  
   - Сохраните найденные файлы в папку `recovered_ntfs`.  

4. **Запуск PhotoRec**  
   - Запустите `photorec`.  
   - Выберите диск, укажите тип файловой системы (`None` – для RAW).  
   - Укажите каталог для сохранения восстановленных файлов.  
   - Позвольте программе сканировать весь диск.  

5. **Повторить для Flash‑накопителя**  
   - Примените те же шаги к накопителю с файловой системой exFAT.  
   - Обратите внимание на работу TRIM: если TRIM активен, восстановление может быть невозможным.  

6. **Анализ результатов**  
   - Сравните количество восстановленных файлов с исходным количеством.  
   - Оцените целостность файлов (проверка контрольных сумм, открытие).  
   - Оформите таблицу с результатами для каждого носителя.  

## Контрольные вопросы / задания для самопроверки  

1. Что такое TRIM и как он влияет на восстановление данных на Flash‑накопителях?  
2. Чем отличается работа TestDisk от PhotoRec?  
3. Какие типы файловых систем поддерживает TestDisk?  
4. Как определить, что файл восстановлен полностью и корректно?  
5. Какие меры предосторожности следует принять перед удалением файлов для успешного восстановления?  
6. Почему иногда восстановление файлов возможно только частично?  
7. Как проверить целостность восстановленного изображения PNG?  
8. Что делать, если после восстановления файлов они открываются с ошибками?  
9. Какие параметры можно задать в PhotoRec для ускорения процесса сканирования?  
10. Как можно защитить данные от потери при работе с Flash‑накопителями?  

## Отчёт по работе  

Структура отчёта:  

1. **Титульный лист** (название работы, дата, ФИО студента, группа).  
2. **Введение** – цель работы, актуальность темы.  
3. **Теоретическая часть** – изложение ключевых понятий.  
4. **Методика выполнения** – пошаговый алгоритм, использованные программы и версии.  
5. **Результаты** – таблица с количеством исходных и восстановленных файлов, график качества.  
6. **Анализ** – обсуждение успешных и неудачных попыток, влияние TRIM, причины потерь.  
7. **Выводы** – основные уроки, рекомендации по защите данных.  
8. **Приложения** – скриншоты логов TestDisk/PhotoRec, копии восстановленных файлов (если разрешено).
