---

## Цель и задачи  
1. Научиться анализировать техническую документацию, выделяя ключевые алгоритмы.  
2. Разработать диагностическое приложение на Python, реализующее выбранный алгоритм.  
3. Провести экспериментальное тестирование программы и проанализировать результаты.  
4. Подготовить отчёт, отражающий процесс разработки и выводы по работе.  

---

## Теоретическая часть  

| Понятие | Описание |
|---------|----------|
| Техническая документация | Набор документов, описывающих структуру, функциональность и требования к программному обеспечению. |
| Диагностический алгоритм | Последовательность действий, позволяющая выявить наличие, характер и степень ошибок в системе. |
| Модульность | Принцип разделения программы на независимые, взаимосвязанные компоненты. |
| Тестирование | Процесс проверки корректности работы программы на основе набора тестовых данных. |
| Метрики качества | Показатели (например, покрытие кода, количество ошибок) измеряющие качество программы. |
| Логирование | Запись событий выполнения программы для последующего анализа. |
| Исключения | События, нарушающие нормальный поток выполнения программы, обрабатываемые специальными блоками. |
| Пакет | Коллекция модулей, объединенных в единый логический блок. |
| Виртуальное окружение | Изолированная среда выполнения, содержащая специфические версии библиотек. |
| Паттерн «Стратегия» | Оформление алгоритмов как объектов, позволяющее менять поведение без изменения клиента. |

## Ход выполнения работы  

1. **Подготовка окружения**  
   - Создайте виртуальное окружение:  
     ```bash
     python -m venv venv
     source venv/bin/activate  # Windows: venv\Scripts\activate
     ```  
   - Установите необходимые библиотеки:  
     ```bash
     pip install numpy pandas
     ```

2. **Изучение технической документации**  
   - Внимательно прочитайте разделы, содержащие описание алгоритма диагностики (например, «Алгоритм расчёта индекса стабильности»).  
   - Выделите входные параметры, промежуточные вычисления и окончательный результат.  

3. **Проектирование программы**  
   - Нарисуйте схему модулей:  
     * `input_handler.py` – чтение данных.  
     * `diagnostics.py` – реализация алгоритма.  
     * `logger.py` – логирование.  
   - Определите интерфейсы между модулями.  

4. **Реализация алгоритма**  
   - В файле `diagnostics.py` реализуйте функцию `calculate_stability_index(data)`.  
     ```python
     import numpy as np
     
     def calculate_stability_index(data: np.ndarray) -> float:
         """
         Рассчитывает индекс стабильности по формуле:
         S = (max(data) - min(data)) / mean(data)
         """
         max_val = np.max(data)
         min_val = np.min(data)
         mean_val = np.mean(data)
         return (max_val - min_val) / mean_val
     ```
   - Добавьте обработку исключений и логирование.  

5. **Создание тестовых данных**  
   - Сгенерируйте массивы случайных чисел с помощью `numpy.random`.  
     ```python
     import numpy as np
     
     test_data = np.random.normal(loc=50, scale=5, size=1000)
     ```

6. **Тестирование программы**  
   - Напишите модульные тесты в файле `test_diagnostics.py` с использованием `unittest`.  
     ```python
     import unittest
     from diagnostics import calculate_stability_index
     
     class TestDiagnostics(unittest.TestCase):
         def test_basic(self):
             data = [10, 20, 30]
             self.assertAlmostEqual(calculate_stability_index(np.array(data)), 1.0)
     
     if __name__ == "__main__":
         unittest.main()
     ```
   - Запустите тесты: `python -m unittest test_diagnostics.py`.  

7. **Экспериментальная часть**  
   - Запустите программу на реальных данных (например, измерения температуры).  
   - Сохраните результаты в файл `results.csv`.  
   - Сравните результаты с ожидаемыми, оцените точность.  

8. **Анализ результатов**  
   - Постройте график зависимости индекса стабильности от времени.  
   - Оцените влияние параметров входных данных на результат.
