# Лабораторная работа № 2  
Использование специального оборудования для эксплуатационного обслуживания мобильных устройств

## Цель и задачи  
1. Познакомиться с принципами работы специализированного диагностического оборудования для мобильных устройств.  
2. Научиться подключать и конфигурировать оборудование через Python‑скрипты.  
3. Разработать простую программу, которая собирает данные о состоянии батареи, памяти и сети и сохраняет их в CSV‑файл.  
4. Проанализировать полученные данные и сформировать рекомендации по обслуживанию.

## Теоретическая часть  
- **Диагностическое оборудование** – устройство, позволяющее получать внутренние данные мобильного телефона (лог, состояние компонентов, параметры сети).  
- **UART/USB** – протоколы передачи данных от оборудования к компьютеру.  
- **Python‑сокеты** – библиотека `socket` для работы с TCP/UDP соединениями.  
- **JSON** – формат обмена данными между оборудованием и скриптом.  
- **CSV** – формат таблицы, удобный для дальнейшего анализа.  
- **Батарея Li‑Ion** – тип аккумулятора, используемый в большинстве смартфонов. Ключевые параметры: напряжение, ёмкость, циклы разрядки.  
- **Память RAM** – оперативная память, её состояние отражает работоспособность системы.  
- **Сеть (Wi‑Fi, LTE)** – параметры соединения (сигнал, скорость, задержка).  
- **Операционная система** – версия и тип (Android, iOS, Windows Mobile) влияют на доступ к API диагностики.  
- **Методология эксплуатации** – план обслуживания, частота проверок, критерии отклонения.  
- **Гибкое программирование** – использование функций, классов и модуля `argparse` для настройки скрипта.  
- **Обработка ошибок** – блоки `try/except`, логирование ошибок через `logging`.  
- **Тестирование** – проверка корректности данных, сравнение с эталонными значениями.  

## Ход выполнения работы  

1. **Подготовка оборудования**  
   - Включить диагностическое устройство.  
   - Подключить его к компьютеру через USB.  
   - Установить драйверы и убедиться, что устройство видится в системе (`ls /dev/tty*` или `wmic path Win32_PnPEntity`).  

2. **Установка необходимых библиотек**

3. **Создание класса `DeviceConnector`** – отвечает за подключение и чтение данных.  
   ```python
   import serial
   import json

   class DeviceConnector:
       def __init__(self, port, baudrate=115200, timeout=1):
           self.ser = serial.Serial(port, baudrate, timeout=timeout)

       def read_data(self):
           line = self.ser.readline().decode().strip()
           return json.loads(line)
   ```

4. **Функция `collect_metrics`** – собирает показатели батареи, памяти и сети.  
   ```python
   def collect_metrics(connector, iterations=5):
       data = []
       for _ in range(iterations):
           metrics = connector.read_data()
           data.append(metrics)
       return data
   ```

5. **Сохранение результатов в CSV**  
   ```python
   import pandas as pd

   def save_to_csv(data, filename='metrics.csv'):
       df = pd.DataFrame(data)
       df.to_csv(filename, index=False)
   ```

6. **Основной скрипт**  
   ```python
   import argparse

   def main():
       parser = argparse.ArgumentParser(description='Сбор данных диагностики')
       parser.add_argument('--port', required=True, help='Порт подключения устройства')
       args = parser.parse_args()

       conn = DeviceConnector(args.port)
       metrics = collect_metrics(conn)
       save_to_csv(metrics)
       print('Данные сохранены в metrics.csv')

   if __name__ == '__main__':
       main()
   ```

7. **Запуск скрипта и проверка результата**  
   ```bash
   python collect_metrics.py --port /dev/ttyUSB0
   ```
   Откройте `metrics.csv` в Excel или LibreOffice, проверьте наличие всех полей.

8. **Анализ данных**  
   - Рассчитать среднее напряжение батареи, максимальную загрузку памяти, среднюю скорость сети.  
   - Сравнить с нормативными значениями, указанными в документации к оборудованию.  

9. **Выводы и рекомендации**  
   - Если напряжение ниже порога, предложить заменить батарею.  
   - При высокой загрузке памяти – очистить кэш или обновить ОС.  
   - При низкой скорости сети – проверить настройки APN или провести диагностику сети.
