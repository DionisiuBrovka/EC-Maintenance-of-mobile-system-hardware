# Практическая работа № 4  
## Алгоритмы проведения схемного контроля  

### Цель и задачи  
1. Научиться формулировать и реализовывать алгоритмы схемного контроля.  
2. Освоить методы проверки корректности схемы на основе логических выражений.  
3. Разработать программу, выполняющую автоматическую проверку схемы.  
4. Оформить отчёт, демонстрирующий применение теоретических знаний на практике.  

### Теоретическая часть  

| Понятие | Описание | Формула / прим. |
|---------|----------|----------------|
| Схема контроля | Набор логических элементов, соединенных в цепь, предназначенный для проверки корректности работы другого устройства. | – |
| Логический элемент | Блок, реализующий базовую логическую операцию (AND, OR, NOT, XOR, NAND, NOR). | – |
| Сигнал | Входной/выходной поток, принимающий бинарные значения 0/1. | – |
| Условие выполнения | Логическое выражение, истинность которого определяет правильность работы схемы. | `S = A AND B OR NOT C` |
| Алгоритм проверки | Последовательность действий, определяющих, как проверяется схема: инициализация, генерация тестовых векторов, измерение выходов, сравнение с ожидаемыми. | – |
| Тестовый вектор | Конкретный набор значений входов, подаваемых на схему. | `V = (A=1, B=0, C=1)` |
| Таблица истинности | Перечень всех возможных комбинаций входов и соответствующих выходов. | – |
| Ошибка схемы | Отклонение фактического выхода от ожидаемого. | `E = |S_actual – S_expected|` |
| Алгоритм генерации | Метод создания тестовых векторов (полный перебор, случайный, на основе анализа). | – |
| Алгоритм сравнения | Процесс проверки каждого выхода схемы с ожидаемым значением. | – |

### Ход выполнения работы  

1. **Определить схему контроля**  
   - Выбрать набор логических элементов и их соединения.  
   - Составить схему в виде графа.  
   - Пример схемы:  
     ```text
     A ──┐
         │
         AND ──┐
     B ──┘      │
                OR ── S
     C ──NOT─────┘
     ```

2. **Составить таблицу истинности**  
   - Перечислить все комбинации входов `A, B, C` (2³ = 8 вариантов).  
   - Рассчитать ожидаемый выход `S`.  
   - Пример таблицы:  

     | A | B | C | S_expected |
     |---|---|---|-------------|
     | 0 | 0 | 0 | 1 |
     | 0 | 0 | 1 | 0 |
     | … | … | … | … |

3. **Разработать программу на Python**  
   - Создать функцию `compute_s(a,b,c)` возвращающую выход схемы.  
   - Создать функцию `generate_vectors()` генерирующую все тестовые векторы.  
   - Создать функцию `check_vectors()` сравнивающую реальные выходы с ожидаемыми.  

   ```python
   def compute_s(a, b, c):
       """Построение схемы: AND + OR + NOT."""
       return (a and b) or (not c)

   def generate_vectors():
       """Генерация всех 8 векторов."""
       for a in (0, 1):
           for b in (0, 1):
               for c in (0, 1):
                   yield (a, b, c)

   def check_vectors():
       for a, b, c in generate_vectors():
           expected = (a and b) or (not c)
           actual = compute_s(a, b, c)
           print(f"{a}{b}{c} → {actual} (expected: {expected})")
   ```

4. **Запустить программу и проанализировать результаты**  
   - Проверить, что все выводы совпадают с таблицей истинности.  
   - При обнаружении несоответствия проанализировать возможные ошибки в схеме.  

5. **Составить отчёт**  
   - Включить схемы, таблицы, код программы, результаты проверки и выводы.
