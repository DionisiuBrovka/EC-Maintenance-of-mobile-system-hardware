Работа с технической документацией для схемного контроля  

## Цель и задачи  
1. Ознакомиться с основными элементами технической документации, применяемой в схемном контроле.  
2. Научиться извлекать и интерпретировать ключевую информацию из чертежей и спецификаций.  
3. Разработать простое Python‑скрипт, позволяющий автоматически проверять соответствие схемы заданным параметрам.  
4. Оформить отчёт, демонстрирующий процесс анализа и полученные результаты.  

## Теоретическая часть  

| Понятие | Краткое определение |
|---------|---------------------|
| Техническая документация | Набор документов (чертежи, схемы, спецификации), содержащих сведения о конструкции и параметрах изделия. |

## Ход выполнения работы  

1. **Сбор и подготовка документации**  
   - Найдите в библиотеке чертежи и спецификации выбранного изделия (например, датчик температуры).  
   - Скопируйте изображения в папку проекта и сохраните их как `drawing.png`.  
   - Сохраните спецификацию в текстовом файле `spec.txt`.  

2. **Анализ требований**  
   - Изучите ГОСТ 2.101‑2014, чтобы понять, какие параметры необходимо проверять (длина, ширина, материал, допуск).  
   - Вспомните, что в спецификации обычно перечислены параметры в формате `Параметр: Значение ± Допуск`.  

3. **Разработка парсера спецификации**  
   ```python
   import re
   from typing import Dict, Tuple

   def parse_spec(file_path: str) -> Dict[str, Tuple[float, float]]:
       """
       Читает файл spec.txt и возвращает словарь вида
       { 'Параметр': (значение, допуск), ... }
       """
       spec = {}
       pattern = re.compile(r'(?P<name>[\w\s]+):\s*(?P<value>[\d\.]+)\s*±\s*(?P<tolerance>[\d\.]+)')
       with open(file_path, 'r', encoding='utf-8') as f:
           for line in f:
               match = pattern.search(line)
               if match:
                   name = match.group('name').strip()
                   value = float(match.group('value'))
                   tolerance = float(match.group('tolerance'))
                   spec[name] = (value, tolerance)
       return spec
   ```
   - **Пояснение**: регулярное выражение ищет строки вида `Параметр: 12.5 ± 0.2`.  
   - **Пример**: если в файле строка `Ширина: 30.0 ± 0.5`, то в словаре будет `{'Ширина': (30.0, 0.5)}`.  

4. **Извлечение параметров из изображения**  
   - Используйте библиотеку `pytesseract` для OCR.  
   ```python
   import pytesseract
   from PIL import Image

   def extract_text(image_path: str) -> str:
       img = Image.open(image_path)
       return pytesseract.image_to_string(img, lang='rus')
   ```
   - **Пояснение**: функция возвращает текст, распознанный на чертеже.  
   - **Пример**: из изображения с пометкой `Ширина 30.0` вы получите строку `"Ширина 30.0"`.

5. **Сравнение данных**  
   ```python
   def compare_specs(spec: Dict[str, Tuple[float, float]], extracted: str) -> Dict[str, bool]:
       """
       Сравнивает значения из spec с текстом extracted.
       Возвращает словарь { 'Параметр': True/False }.
       """
       results = {}
       for name, (value, tol) in spec.items():
           pattern = re.compile(rf'{name}\s*[:\s]*([\d\.]+)')
           match = pattern.search(extracted)
           if match:
               measured = float(match.group(1))
               results[name] = abs(measured - value) <= tol
           else:
               results[name] = False
       return results
   ```
   - **Пояснение**: ищет в тексте число после названия параметра и сравнивает с допустимым диапазоном.  

6. **Запуск и вывод результатов**  
   ```python
   if __name__ == '__main__':
       spec = parse_spec('spec.txt')
       text = extract_text('drawing.png')
       comparison = compare_specs(spec, text)
       for param, ok in comparison.items():
           status = 'OK' if ok else 'FAIL'
           print(f'{param}: {status}')
   ```
   - **Пояснение**: программа выводит статус проверки каждого параметра.

## Отчёт по работе  

Структура отчёта должна включать:

1. **Титульный лист** – название работы, курс, имя студента, преподаватель, дата.  
2. **Введение** – цель работы, её актуальность и основные задачи.  
3. **Теоретическая часть** – краткое изложение ключевых понятий (таблица из раздела «Теоретическая часть»).  
4. **Методика** – описание выбранного подхода, используемых инструментов (Python, pytesseract, регулярные выражения).  
5. **Результаты** – таблица, где для каждого параметра указано: номинальное значение, допуск, измеренное значение, статус проверки.  
6. **Обсуждение** – анализ полученных результатов, возможные источники ошибок, рекомендации по улучшению.  
7. **Выводы** – основные выводы о соответствии схемы требованиям и об эффективности автоматической проверки.  
8. **Приложения** – исходный код скриптов, скриншоты работы, копии спецификации и чертежа.
