# Лабораторная работа № 3  
Использование сигнатурного анализатора для эксплуатационного обслуживания мобильных систем

## Цель и задачи

1. Познакомиться с принципами работы сигнатурного анализа кода и его ролью в обеспечении безопасности мобильных приложений.  
2. Научиться использовать инструменты Python для генерации и сравнения сигнатур программных компонентов.  
3. Оценить эффективность обнаружения уязвимостей при помощи сигнатурного анализа.  
4. Разработать практический прототип, способный автоматически выявлять подозрительные участки кода в мобильном приложении.

## Теоретическая часть

| Понятие | Описание |
|---------|----------|
| **Сигнатура** | Уникальное представление сущности (функции, класса, модуля), обычно в виде хеш‑значения. |
| **Хеш‑функция** | Алгоритм, преобразующий произвольные данные в фиксированную длину. |
| **SHA‑256** | Один из наиболее распространённых криптографических хеш‑алгоритмов. |
| **Криптографический хеш** | Хеш, удовлетворяющий свойствам коллизий‑устойчивости и непредсказуемости. |
| **Метод «фингерпринт»** | Создание короткой подписи (фингерпринта) кода для быстрого сравнения. |
| **Сигнатурный анализ** | Процесс сравнения сигнатур кода с базой известных паттернов для обнаружения вредоносных или некорректных участков. |
| **База сигнатур** | Коллекция хеш‑значений известных паттернов (уязвимости, вредоносный код). |
| **Static Analysis** | Статический анализ кода без его выполнения. |
| **Dynamic Analysis** | Динамический анализ кода во время выполнения. |
| **Кросс‑платформенность** | Возможность работы инструмента на разных ОС (Windows, Linux, macOS). |

**Формула**:  
\[
\text{Signature} = \text{Hash}(\text{NormalizedCode})
\]
где `NormalizedCode` – код после удаления комментариев, пробелов и форматирования.

## Ход выполнения работы

1. **Подготовка среды**  
   - Установить Python 3.11 (или выше).  
   - Установить библиотеки: `pip install hashlib` (встроенный), `pip install pycryptodome` для более сложных хешей.  
   ```python
   import hashlib
   ```

2. **Извлечение кода из мобильного проекта**  
   - Скопировать исходный код проекта (Android‑Java, Kotlin или iOS‑Swift).  
   - Конвертировать каждый файл в UTF‑8, удалить комментарии и пустые строки.  
   ```python
   def normalize_code(text: str) -> str:
       import re
       text = re.sub(r'//.*|/\*[\s\S]*?\*/', '', text)  # удаляем комментарии
       text = re.sub(r'\s+', ' ', text)  # заменяем множественные пробелы
       return text.strip()
   ```

3. **Генерация сигнатур**  
   - Для каждого файла вычислить SHA‑256.  
   ```python
   def file_signature(path: str) -> str:
       with open(path, 'r', encoding='utf-8') as f:
           code = normalize_code(f.read())
       return hashlib.sha256(code.encode()).hexdigest()
   ```

4. **Создание базы сигнатур**  
   - Сохранить сигнатуры в CSV‑файл `signatures.csv` со столбцами: `file_path,signature`.  
   ```python
   import csv
   with open('signatures.csv', 'w', newline='') as csvfile:
       writer = csv.writer(csvfile)
       writer.writerow(['file_path', 'signature'])
       for path in file_list:
           writer.writerow([path, file_signature(path)])
   ```

5. **Сравнение с известными паттернами**  
   - Загрузить базу известных уязвимостей (`known_signatures.csv`).  
   - Для каждого сигнатуры проекта проверить наличие в известной базе.  
   ```python
   known = set()
   with open('known_signatures.csv') as f:
       for row in csv.reader(f):
           known.add(row[1])

   for path, sig in project_signatures:
       if sig in known:
           print(f'Уязвимость найдена в {path}')
   ```

6. **Анализ результатов**  
   - Составить отчёт о найденных совпадениях: путь файла, тип уязвимости, вероятность.  
   - Оценить точность: процент совпадений / общее число файлов.  

7. **Экспериментальная часть**  
   - Добавить в проект искусственно известный уязвимый фрагмент.  
   - Повторить шаги 2‑6 и убедиться, что система обнаружила добавленный код.
