# Лабораторная работа № 6  
**Тестирование памяти с помощью программных методов контроля**

## Цель и задачи  
1. Изучить принципы работы оперативной памяти и влияние программных ошибок на её состояние.  
2. Научиться использовать встроенные средства Python для измерения потребления памяти (модуль `tracemalloc`, `gc`, `psutil`).  
3. Разработать программу, позволяющую выявлять утечки памяти и измерять используемую память в реальном времени.  
4. Оценить влияние различных алгоритмов и структур данных на потребление памяти.

## Теоретическая часть  
| Понятие | Определение | Примечание |
|---------|-------------|------------|
| **Оперативная память (RAM)** | Быстрый тип памяти, в которой хранятся данные, необходимые процессору в данный момент. | Время доступа ~ 10‑100 нс |
| **Утечка памяти** | Неосвобожденные блоки памяти, которые больше не нужны, но остаются в резерве. | В Python обычно связано с сильными ссылками в циклах |
| **Сборщик мусора (GC)** | Автоматический механизм освобождения памяти, обнаруживающий неиспользуемые объекты. | В CPython использует подсчёт ссылок и циклическую сборку |
| **`tracemalloc`** | Модуль Python, позволяющий отслеживать аллокации памяти и сравнивать их по времени. | Устанавливается с `import tracemalloc` |
| **`gc`** | Модуль Python для работы с сборщиком мусора: включение/выключение, ручной запуск. | `gc.collect()` принудительно запускает сборку |
| **`psutil`** | Библиотека для получения информации о системе, включая потребление памяти процессом. | `psutil.Process().memory_info().rss` возвращает RSS |
| **RSS (Resident Set Size)** | Физическая память, занятую процессом в данный момент. | Не включает память, находящуюся в свопе |
| **Память за пределами Python** | Модули, написанные на C/C++ и использующие собственные аллокации. | Их можно отслеживать через `psutil` или профайлеры |
| **Фрагментация памяти** | Распределённое использование памяти, приводящее к неэффективному использованию. | В Python фрагментация минимальна благодаря пулу аллокаций |
| **Алгоритмы экономии памяти** | Использование генераторов, `__slots__`, `array.array`, `numpy` вместо списков. | Снижает потребление RAM, но может влиять на производительность |

## Ход выполнения работы  

1. **Подготовка окружения**  
   - Установить Python 3.10+ и пакеты `psutil`.  
   ```bash
   pip install psutil
   ```  
   - Создать пустой проект в IDE/текстовом редакторе.

2. **Изучение утилит `tracemalloc`**  
   - Включить трассировку и собрать статистику после выполнения кода.  
   ```python
   import tracemalloc
   tracemalloc.start()
   # код для тестирования
   snapshot = tracemalloc.take_snapshot()
   top_stats = snapshot.statistics('lineno')
   for stat in top_stats[:10]:
       print(stat)
   ```
   - Пояснение: `tracemalloc` хранит только 1 МБ истории аллокаций, поэтому важно использовать `take_snapshot` в нужный момент.

3. **Проверка использования памяти процессом**  
   - С помощью `psutil` получить RSS до и после выполнения блока кода.  
   ```python
   import psutil, os
   process = psutil.Process(os.getpid())
   rss_before = process.memory_info().rss
   # код
   rss_after = process.memory_info().rss
   print(f'Memory delta: {rss_after - rss_before} bytes')
   ```

4. **Создание сценария с потенциальной утечкой**  
   - Написать функцию, создающую объекты в цикле без удаления ссылок.  
   ```python
   def leak():
       data = []
       for i in range(10000):
           data.append([i] * 1000)  # создаём большие списки
   ```
   - Запустить функцию и измерить память при помощи шагов 2 и 3.  

5. **Очистка памяти**  
   - Принудительно вызвать сборщик мусора после выполнения функции.  
   ```python
   import gc
   gc.collect()
   ```
   - Сравнить результаты до и после `gc.collect()`.

6. **Анализ результатов**  
   - Составить таблицу:  
     | Метрика | Значение до | Значение после | Δ | Примечание |
     |---------|-------------|----------------|---|------------|
   - Вывести выводы: наличие утечки, эффективность `gc`, влияние `tracemalloc`.

7. **Оптимизация кода**  
   - Переписать функцию, используя генераторы и `__slots__`.  
   - Сравнить потребление памяти после оптимизации.

8. **Формирование отчёта**  
   - Собрать результаты, графики (можно использовать `matplotlib` для визуализации).  
   - Приложить исходный код и снимки статистики.
